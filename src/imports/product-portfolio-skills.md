Redesign the Skills section so it feels more like a product portfolio section and less like a resume list.

Current issues:

* too many pills
* flat hierarchy
* feels like a CV block instead of a narrative section

Design goals:

* communicate the intersection of product, design and engineering
* create visual hierarchy
* keep the design minimal and premium

Layout:

Large section title:
"Working across product, design and engineering."

Below the title create three skill groups:

Product
Design
Technical

Each group should contain only 4–5 key skills.

Reduce the number of pills and increase spacing.

Each group should feel like a mini block rather than a long horizontal list.

Visual improvements:

* increase vertical spacing between groups
* slightly larger group titles
* softer pill design
* subtle hover state on skills
* staggered reveal animation when the section enters the viewport

Optional improvement:

Add a subtle visual hint that these three areas intersect.

For example:

* connecting lines
* layout overlap
* or a highlighted center area suggesting "product engineering".

Overall goal:

The section should feel like a statement about multidisciplinary product work, not a checklist of skills.
